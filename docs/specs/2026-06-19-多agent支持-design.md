# rookie-work 多 agent 支持 设计 spec（Codex + Hermes）

> 在已发布的 Claude Code 插件（v1.0.0）之上，新增 OpenAI Codex 与 NousResearch Hermes(`hermes-agent`) 支持。
> 状态：设计定稿（2026-06-19），待写实现计划。前置 spec：`2026-06-19-rookie-work-design.md`（基础设计）。

## 0. 一句话

让 rookie-work 的同一套「方法正文」在 Claude Code、Codex、Hermes 上都生效，且 **Codex 与 Hermes 都守住两条核心保证：①强制每会话注入纪律、②持久（跨会话）关闭开关**——靠各 agent 的脚本钩子 + 统一的标记文件关闭法实现。

## 1. 背景与目标

- **现状**：rookie-work v1.0.0 是 Claude Code 插件，已自托管发布于 `github.com/xuzheng1210/rookie-work`。三件套：常驻纪律（SessionStart 钩子注入 `SESSION-PREAMBLE.md`）+ 可调用技能（`SKILL.md` → `/rookie-work`）+ 标记文件关闭开关。
- **目标**：新增 Codex、Hermes 支持，**两家都保住强制注入 + 持久关闭**（这是 rookie-work 区别于"纯技能"插件的命根，已由用户拍定）。
- **可行性（发现阶段已确认）**：三家都用「SKILL.md（`name`+`description` 已合规）+ 脚本钩子注入 + 标记文件」这套模型；核心正文/preamble/references/关闭法三家通用。差异只在外壳（技能位置、清单格式、钩子事件与输出、安装渠道）。

## 2. 架构：CC 根 = 单一真相源，build 脚本生成另两家

```
rookie-work/                         # 仓库根 = CC 插件（真相源；published；基本不动）
  SKILL.md  SESSION-PREAMBLE.md  references/   # ← 唯一权威内容
  .claude-plugin/{plugin,marketplace}.json
  hooks/{hooks.json, session-start, run-hook.cmd}
  tests/{test-session-start.sh, test-skill.sh}
  build/
    build.sh                         # 读权威内容 → 装配 Codex / Hermes 包
    templates/                       # 各 agent 的清单/钩子/config 模板
  dist/                              # ← 生成物：提交入库、带「勿手改」头注
    codex/     # .codex-plugin/plugin.json + skills/rookie-work/{SKILL.md,references/} + hooks/
    hermes/    # skills/rookie-work/{SKILL.md,references/} + agent-hooks/rookie-work-inject.sh + config-snippet.yaml + INSTALL.md
  docs/{specs,plans}/
  README.md                          # + Codex / Hermes 安装段
```

- **build.sh（bash，开发期工具）**：读仓库根的权威内容，装配出 `dist/codex/`、`dist/hermes/`。生成文件首行加 `# GENERATED by build/build.sh — DO NOT EDIT`。
- **生成物提交入库**：自托管安装需从仓库取到，故 `dist/` 提交（非 gitignore）。
- **防漂移闸**：一个测试重跑 `build.sh` 后断言 `dist/` 无 `git diff`——改了权威内容忘重跑就红，把「单一真相源」变成强制约束。
- **CC 不动**：仓库根的 CC 插件与 v1.0.0 安装路径（`marketplace.json` `source:"./"`）保持不变。

## 3. 共享 vs 各 agent 差异

**三家逐字相同（build 原样拷贝）**：`SKILL.md` 正文、`SESSION-PREAMBLE.md`、`references/{changelog-format,model-and-review-policy}.md`。

**钩子两变体：**
- **CC + Codex 共用一个 `session-start`**：输出 `{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":…}}`，读 `SESSION-PREAMBLE.md`，查标记文件。Codex 与 CC 契约逐字相同、且提供 `CLAUDE_PLUGIN_ROOT`/`CLAUDE_PLUGIN_DATA` 别名，故脚本可直接复用。**唯一需改（权威源一处）**：项目目录变量改为 `PROJECT_DIR="${CODEX_PROJECT_DIR:-${CLAUDE_PROJECT_DIR:-$PWD}}"`（对 CC 无害，使 Codex 复用）。
- **Hermes 单独 `agent-hooks/rookie-work-inject.sh`**：`pre_llm_call` 钩子，stdout 输出 `{"context":…}`；读 stdin payload 的 `extra.is_first_turn`，仅为 true 时注入 preamble、否则输出 `{}`（把"每回合触发"收敛到"只开场"）；同一套标记文件检查（`[ -f ~/.rookie-work-off ] || [ -f <project>/.rookie-work-off ]` 则输出 `{}`）。

**清单各一份：**
- CC：`.claude-plugin/{plugin,marketplace}.json`（现状）。
- Codex：`.codex-plugin/plugin.json`（必填 `name`/`version`/`description` + `"skills":"./skills/"`；可选 `interface`{displayName:"Rookie Work",category}）。技能置于 `skills/rookie-work/SKILL.md`（Codex 要求子目录，非根）。
- Hermes：技能 frontmatter 即清单（`name`+`description`，可选 `metadata.hermes.{tags,homepage}`）；钩子靠 `config.yaml` 的 `hooks: { pre_llm_call: [...] }` 片段声明（由 `dist/hermes/config-snippet.yaml` + INSTALL 文档提供）。

## 4. 各 agent 包规格

### 4.1 Codex 包（`dist/codex/`）
- `.codex-plugin/plugin.json`（清单，见上）。
- `skills/rookie-work/SKILL.md`（= 权威 SKILL.md，frontmatter 不变）+ `skills/rookie-work/references/*`。
- `hooks/{hooks.json, session-start, run-hook.cmd}`（= CC 钩子；`hooks.json` matcher `startup|resume|clear|compact`）。
- 调用：`$rookie-work` 或 `/skills`；description 也支持隐式自动加载。

### 4.2 Hermes 包（`dist/hermes/`）
- `skills/rookie-work/SKILL.md`（= 权威 SKILL.md）+ `references/*`。Hermes 按 `name` slug 自动暴露 `/rookie-work`。
- `agent-hooks/rookie-work-inject.sh`（`pre_llm_call` 钩子，见上）。
- `config-snippet.yaml`：要用户合并进 `~/.hermes/config.yaml` 的 `hooks:` 片段。
- `INSTALL.md`：放技能目录 + 加钩子 + 首次同意说明。

## 5. 安装（自托管，三家从 GitHub 仓库；官方市场列 backlog）

- **CC**：不变——`/plugin marketplace add https://github.com/xuzheng1210/rookie-work` → `/plugin install rookie-work@rookie-work-marketplace`。
- **Codex**：自托管 Codex marketplace（若 schema 实测可行），或把 `dist/codex/skills/rookie-work/` 拷进 `~/.codex/skills/` 并按 `dist/codex/` 的 INSTALL 装钩子。
- **Hermes**：`hermes skills tap add xuzheng1210/rookie-work`（若实测能发现 `dist/hermes` 技能），或把 `dist/hermes/skills/rookie-work/` 拷进 `~/.hermes/skills/` + 合并 `config-snippet.yaml` 的钩子（首次同意，或 `HERMES_ACCEPT_HOOKS=1`/`hooks_auto_accept`）。
- README 给每家确切命令；确切自托管接线以"先验证"任务结果为准（见 §7）。

## 6. 关闭开关

- **三家统一**：标记文件 `~/.rookie-work-off`（全局）/ `<project>/.rookie-work-off`（项目），由各 agent 的钩子脚本检查、存在即不注入纪律。
- **Hermes 额外**：原生 `skills.disabled:[rookie-work]`（从斜杠命令表 + 系统提示索引一并过滤）/ 删 `config.yaml` 钩子条目，README 一并写。
- SKILL.md 的"口头当场关 / 持久关"一节三家通用（持久关的标记路径已与钩子逐字一致）。

## 7. 风险 / 待实测（明确风险；每个 agent 的实现计划第一项任务 = 真机安装验证兜底，验证通过才继续）

1. **【高】Codex 插件自带 `hooks/hooks.json` 是否安装即自动激活**：官方文档说可 bundle，但 superpowers 的 Codex 包刻意剔除了 hooks，存在矛盾。**先验证**；若不自动激活，退路 = 文档教用户加用户级 `~/.codex/hooks.json`（已确认支持），仍保住强制注入。
2. **【中】Hermes `pre_llm_call` 每回合注入的体验/成本 + 钩子首次同意对新手的摩擦**：**先验证** `is_first_turn` 收敛是否够、同意流程是否可接受；摩擦过大则在 INSTALL 里给 `hooks_auto_accept`/env 的清晰指引。
3. **【中】Codex `CODEX_PROJECT_DIR` 是否存在**：已用 `$PWD` 兜底；实测确认。
4. **【低】Hermes `claude-marketplace` 源能否直接吃 `.claude-plugin/marketplace.json`**：能则多一条安装路；否则走 tap / 丢目录。

## 8. 测试策略

- **build.sh**：跑通；生成物文件齐；各清单合法（JSON/YAML）；**`dist/*/.../SKILL.md` 正文与权威源逐字一致**（防漂移）。
- **防漂移闸**：CI/测试重跑 `build.sh` 后 `git diff --exit-code dist/` 必须干净。
- **钩子单测**：Codex 复用 CC 的 `test-session-start.sh`（隔离 HOME，验证默认注入/全局关/项目关）；Hermes 新增 `test-pre-llm-call.sh`（首回合注入 / 非首回合空 / 标记抑制）。
- **真机冒烟**：每个 agent 的实现计划以"先真机验证"开场（最小带钩子样例，确认技能加载 + 注入 + 关闭），末尾走完整行为冒烟（Tier 0/1/2 + 开关），如 CC。

## 9. 非目标 / backlog

- Hermes 的 **Python 插件路线（Route B）** —— 不做（已选 Route A）。
- **官方市场上架**（openai/plugins PR、Hermes Skills Hub）—— backlog。
- **其它 agent**（Cursor / Gemini / OpenCode 等）—— backlog（架构已为之留出"加一个外壳模板 + 钩子变体"的扩展位）。
- Hermes 的 **SOUL.md 被动常开路线** —— 仅作"轻量默认"记录，非主方案。

## 10. 成功标准

- `build.sh` 从单一真相源生成 Codex、Hermes 包，防漂移闸保证不漂。
- Codex、Hermes 各自：技能可调用（`/rookie-work` 或 `$rookie-work`）；**新会话/首回合自动注入纪律**（强制注入成立，经真机验证）；建/删标记文件可**持久开关**（经真机验证）。
- CC v1.0.0 不受影响。
- 三家共享同一份方法正文，逐字一致。
